SQL Injection Demo

This demo shows a vulnerable login form that is susceptible to SQL injection.

Vulnerable Code

❌ Vulnerable
// This code is vulnerable to SQL injection query = "SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'" result = db.execute(query)

Attack Example

// An attacker could input: username = "' OR '1'='1" password = "' OR '1'='1" // Resulting query: query = "SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '' OR '1'='1'"

Secure Code

✅ Secure
// Using parameterized queries prevents injection query = "SELECT * FROM users WHERE username = ? AND password = ?" result = db.execute(query, (username, password))
Key Takeaway: Always use parameterized queries or an ORM. Never concatenate user input directly into SQL queries.

Cross-Site Scripting (XSS) Demo

This demo shows how XSS occurs when user input is rendered without sanitization.

Vulnerable Code

❌ Vulnerable
<!-- User input is rendered directly --> <div>Welcome, {{ user_input }}</div>

Attack Example

// An attacker could input: user_input = "<script>alert('XSS')</script>" // Resulting HTML: <div><script>alert('XSS')</script></div>

Secure Code

✅ Secure
// Using auto-escaping or sanitization <div>Welcome, {{ escape(user_input) }}</div> // Result: Welcome, &lt;script&gt;alert('XSS')&lt;/script&gt;
Key Takeaway: Always escape user input when rendering in HTML. Use Content Security Policy (CSP) headers for additional protection.

CSRF Attack Demo

This demo shows how CSRF attacks work and how to prevent them.

Vulnerable Form

❌ Vulnerable
<!-- No CSRF protection --> <form action="/transfer" method="POST"> <input type="hidden" name="amount" value="1000"> <input type="hidden" name="to" value="attacker"> <button type="submit">Transfer</button> </form>

Attack Example

<!-- Malicious site makes the request --> <img src="https://bank.com/transfer?amount=1000&to=attacker">

Secure Code

✅ Secure
<!-- With CSRF token --> <form action="/transfer" method="POST"> <input type="hidden" name="csrf_token" value="{{ csrf_token() }}"> <input type="hidden" name="amount" value="1000"> <input type="hidden" name="to" value="recipient"> <button type="submit">Transfer</button> </form>
Key Takeaway: Always include CSRF tokens in forms that perform state-changing actions. Use SameSite cookies for additional protection.

Security Headers Demo

This demo shows the difference between a site with and without proper security headers.

Missing Headers

❌ Vulnerable
# Response headers Content-Type: text/html Server: nginx # Missing security headers!

Secure Headers

✅ Secure
# Complete security headers Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' Strict-Transport-Security: max-age=31536000; includeSubDomains X-Frame-Options: DENY X-Content-Type-Options: nosniff Referrer-Policy: strict-origin-when-cross-origin Permissions-Policy: geolocation=(), microphone=(), camera=()
Key Takeaway: Implement all major security headers to protect against XSS, clickjacking, MIME sniffing, and other attacks.
